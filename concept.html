Conceitos e Ferramentas

Conceitos

Com uma arquitetura que separa os planos de controle e dados, as SDN permitem que o controle da rede seja diretamente programável e que a infra-estrutura subjacente seja abstraída para o acesso a partir de aplicativos.

A ONF (Open Networking Foundation) é uma organização dedicada à promoção e adoção de Redes Definidas por Software através de um padrão livre. São os responsáveis pela introdução do padrão OpenFlow.

[1] propõe uma caracterização das SDNs baseada em quatro pilares:

  • Os planos de dados e de controle são desacoplados. Dispositivos de rede passam a ter o papel de apenas encaminhar pacotes;
  • Decisões de encaminhamento são baseadas no fluxo (fluxo é uma sequência de pacotes entre a fonte e o destino) e não no destino. Todos os pacotes de um fluxo recebem a mesma política nos dispositivos de encaminhamento;
  • A lógica de controle é movida para uma entidade externa, o controlador da SDN ou NOS (Network Operating System) que roda em um servidor e fornece as abstrações e recursos necessários para programar os dispositivos de encaminhamento com base em uma visão centralizada e abstrata da rede;
  • A rede é programável através de aplicações rodando no NOS que interagem com os dispositivos do plano de dados.

Essas funcionalidades são baseadas em três abstrações principais [1], sendo elas:

  • Encaminhamento: Deve permitir qualquer comportamento de encaminhamento à rede sem que seja necessário expor os detalhes do hardware envolvido no processo;
  • Distribuição: Tem como objetivo tornar o problema do controle distribuído em um problema logicamente centralizado. Isso é feito por meio de de uma camada de distribuição comum, que reside no NOS. Essa camada instala os comandos de controle nos dispositivos de encaminhamento e coleta informações do estado da rede;
  • Especificação: Deve possibilitar que uma aplicação de rede expresse o comportamento desejado para a rede sem que seja responsável por implementar esse comportamento, mapeando configurações abstratas especificadas pelas aplicações em uma configuração física na rede global.

Embora no texto o foco principal seja mais relacionado ao OpenFlow, cabe ressaltar que outras formas de implementar SDNs são propostas. Algumas como alternativa ao uso do OpenFlow. Segundo [26], três abordagens atualmente dominam o mercado de SDNs:

  • OpenFlow:
    • Protocolo de comunicação southbound que fornece uma abstração para comunicação entre um controlador e os dispositivos de rede;
    • Baseado em fluxos, permite que a rede seja programada de forma flexível e dinâmica;
    • Necessita de aparelhos compatíveis com seu protocolo no plano de dados;
  • Protocolos de encapsulamento:
    • O exemplo dado na referência é o VXLAN (Virtual eXtensible LAN);
    • Usa uma abordagem de "overlay", para que a SDN possa ser implantada sem que seja necessária a troca da infra-estrutura do plano de dados;
    • Nessa abordagem os pacotes devem ser encapsulados para que seja tunelados para dispositivos inteligentes que os desencapsulam e enviam para o alvo;
    • Tem objetivo de permitir que usuários continuem utilizando sua base de rede;
  • Cisco Application Centric Infrastructure (ACI):
    • É uma solução de infra-estrutura completa executada por um administrador de políticas de rede, que requer dispositivos de rede ineligentes no plano de dados.

Conceitos recorrentes no texto:

  • Hipervisores:
    • Também conhecidos como máquinas virtuais, no caso das aplicações descritas no texto são sistemas que permitem a reprodução de diversas funções de dispositivos de rede em dispositivos genéricos;
  • Interfaces:
    • Northbound: interface utilizada para abstrair os conceitos de baixo nível de uma camada para suas camadas superiores;
    • Southbound: interface utilizada para a comunicação com camadas abaixo de uma certa camada;
  • APIs:
    • Do inglês Application Programming Interface é um conjunto de rotinas, protocolos e ferramentas que fornecem uma interface para facilitar o desenvolvimento de aplicações que interajam com determinado componente de um sistema.

Ferramentas

  • Floodlight
    • Controlador SDN fornecido sob licença Apache e baseado em Java;
    • Trabalha com comutadores físicos e virtuais através do protocolo OpenFlow;
    • Permite a especificação do protocolo a ser utilizado para o controle remoto de dispositivos de rede;
    • É facilmente extensível e fácil de configurar com poucas dependências;
    • Pode administrar redes mistas OpenFlow e não-OpenFlow;
  • Indigo
    • Projeto OpenSource focado em fornecer suporte ao OpenFlow em comutadores físicos e em hipervisores;
    • É a base do projeto Switch Light da Big Switch Networks;
  • Neutron (OpenStack)
    • Parte do projeto OpenStack, fornece a “rede como serviço” (tradução livre de NaaS: networking as a service) entre dispositivos como controladores de interface de rede (NICs) administrados por serviços do OpenStack como o Nova;
    • Permite a criação de topologias de rede complexas na nuvem;
    • Inclui funcionalidades de rede - como Qualidade de Serviço ou Firewalls, por exemplo - por meio de de plugins;
    • Permite a criação serviços de redes avançados acopláveis à redes OpenStack;
    • Interface gráfica para algumas operações;
  • Open vSwitch
    • Open vSwitch é um comutador virtual distribuído de múltiplas camadas, fornecido sob a licença Apache 2.0;
    • Implementa o protocolo OpenFlow;
    • O protocolo OVSDB (Open vSwitch Database Management Protocol) é utilizado, junto à outros controladores, para fornecer informações de configuração ao servidor de banco de dados de comutadores, permitindo assim a manipulação de elementos da topologia interna da implementção Open vSwitch;
  • OpenDaylight
    • [25] "Tem como objetivo fornecer uma plataforma SDN funcional que oferece uma implementação direta de SDN sem a necessidade de outros componentes. Além disso, add-ons podem ser oferecidos para agregar valor."
    • Framework Java com o intuito de acelerar a adoção das SDNs;
    • Suporta OpenFlow assim como outros padrões de SDN;
    • Fornece um controlador para a SDN que expõe APIs northbound abertas;
